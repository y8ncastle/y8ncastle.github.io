HTML 기초 용어 정리 by 생활코딩

2월 15 2020 HTML 9분 소요 (약 1305 단어)



용어 정리

표현 이름 태그 기능
<> - 열리는 표현
</> - 닫히는 표현
<!doctype html> - 웹페이지가 HTML로써 만들어졌다는 것을 표현하기 위해 문서의 시작에 작성하는 표현
<html> - Head와 body 태그를 감싸기로 약속한 하나의 표현
<head> - 문서의 속성을 지정
<title> - 웹 문서의 제목을 설정
<meta charset="utf-8"> - 웹브라우저에게 UTF-8 방식으로 페이지를 열도록 하는 표현
(웹페이지에 저장된 문자 표현 방식과 웹브라우저가 페이지를 해석하는
방식이 다를 때 이상한 문자를 표시하게 되는데, 설정이 없으면 일반적으로
영어가 아닌 문자는 깨져서 나옴)
<body> - 본문 내용 표현
<h1> ~ <h6> Heading 제목 표현 (일반적인 텍스트보다는 글씨가 두껍고 줄바꿈이 됨)
<strong> - 글씨를 진하게 함
<u> Underline 밑줄 표시
<br> - 줄 바꿈 (닫힘 태그 없음)
<p> Paragraph 단락 표현 (하나의 단락을 그룹핑)
<li> List 목차 표현 (ul 태그를 필요로 함)
<ol> Ordered list ul의 정렬된 형태 (자동 넘버링)
<ul> Unordered list li의 부모 태그 (li 태그와 아주 밀접한 관계)
<img src="위치"> - 이미지 표현
<a> Anchor 링크 표현
<iframe src="위치"> - 동영상 표현



속성 정리

속성 (attribute): 태그의 이름만으로는 정보가 부족하여 만들어진 문법


태그 표현 속성명 속성 기능
<img> width=”크기 혹은 퍼센트 비율” 사진의 크기 조정
alt=”내용” 사진이 없을 경우 텍스트 대체
<a> href=”위치” (HyperText Reference) 하이퍼링크 (이 값을 참조)
target=”_blank” 페이지가 새 창에서 열리도록 함
title=”내용” 링크가 어떤 내용을 담고 있는지 툴팁으로 보여줌
<iframe> width=”크기 혹은 퍼센트 비율” 숫자 크기만큼 폭 설정
height=”크기 혹은 퍼센트 비율” 숫자 크기만큼 높이 설정
frameborder=”숫자” 동영상 테두리 두께 설정
allowfullscreen 전체화면 지원



인터넷 및 HTML의 역사

정보사회에서 HTML을 의미에 맞게 잘 사용하는 것은 비즈니스적인 측면에서 매우 중요하다.

웹은 모든 운영체제에서 동작하고, 웹페이지의 소스코드는 누구나 볼 수 있으며, 웹 자체는 저작권이 없는 순수한 공공재이다. 웹의 이러한 특징들이 다른 기술들과 구별되는 특별한 것으로 만든다.

1990년에 시작된 이후로 계속해서 가장 중요한 정보기술로 군림하면서 점점 복잡해졌고, 단순히 지식에 대한 소비자로 살 것이 아니라면 컴퓨터, 인터넷, 웹과 같이 혁신적인 것을 만들기 위해 웹의 역사를 알아야 한다.

웹은 인터넷의 부분집합이며, 웹과 동급인 기술이 별처럼 많이 있기도 하다.

미국에서 1960년대 핵공격이 일어났을 때 통신 시스템에 심각한 취약점이 있다는 것을 발견하고 중앙집중적인 통신 시스템을 분산된 형태의 통신 시스템으로 바꾸는 구상을 하였고, 이는 인터넷의 시작이었다.

수많은 통신 장치들이 각자 일종의 전화국 역할을 하는 상호부조 시스템이 인터넷이며, 중앙은 없다.

또한 웹은 스위스에서 처음으로 탄생하였는데, 제네바의 유럽 입자물리연구소(CERN)라는 저명한 연구소에서 팀 버너스 리에 의해 창시되었다.


세계 최초의 웹 페이지: http://info.cern.ch


인터넷이 동작하기 위해서는 컴퓨터가 최소 2대가 있어야 하며, 1대만 있다면 그것은 인터넷이 아니고, 3대가 있다면 본질적이라고 할 수 없다. 웹 브라우저는 클라이언트에서 동작하며 웹 클라이언트(요청하는 컴퓨터)라고 한다. 반대로 웹 서버는 서버 컴퓨터이며 웹서버(응답하는 컴퓨터)라고 부른다.

Github가 제공하는 여러 기능 중 웹호스팅 기능을 이용해 홈페이지를 운영할 수 있으며, 이는 페이지 기능을 활성화했을 때 동작한다.

HTML은 웹브라우저가 해석하기 때문에 서버에서는 특별히 해줄 일이 없고, 이러한 특성을 정적(static)이라고 한다. 현재는 php, Python, Ruby, Java 등을 통해 동적(dynamic) 웹페이지도 구현할 수 있다.

[2020.02.03-02.05] KCSE 2020 (한국 소프트웨어공학 학술대회)

2월 15 2020 Academy 7분 소요 (약 1051 단어)



2020년 2월 3일부터 5일까지 강원도 평창 한화리조트에서 KCSE 2020(한국 소프트웨어공학 학술대회)이 열렸다.

BaSE 연구실의 연구원들과 함께 2박 3일 MT 가는 느낌으로 다녀오게 되었고, 많은 사람들이 준비한 논문들 중에서 블록체인 위주의 발표를 들으면서 새로운 아이디어를 공유하는 시간을 가진 것 같다. 물론 나는 발표가 없었지만 하하…







등록비가 비싼 행사인만큼 전체적인 분위기라던지, 진행 과정에서 상당히 신경을 많이 쓴 것 같다는 느낌을 받았고 숙소 또한 마음에 들었다.

사진으로는 다 담을 수 없었지만 홀이 정말 컸고 사람들도 엄청 많았다. 또, 대학과 여러 기업에서 참여했는데 코로나 바이러스가 여전히 난리인 상황이어서 마스크 착용과 체온 검사는 필수적인 절차였다.









개회식 이후에는 삼성전자에서 Smart Software Development라는 주제로 기조연설을 하였고, 현재 삼성전자가 어디에 초점을 두고 소프트웨어 분야를 개발하고 있는지, 앞으로 미래에는 어떠한 기술들이 자리 잡게 될 지에 대해 들을 수 있었다.

확실한 건 4차 산업혁명 기술 중에서도 인공지능에 상당한 초점이 맞춰져 있는 것 같았다.

학부 때에도 미래에는 인공지능 기술이 대세가 될 것이라는 얘기를 많이 들었는데 관련 업계에서 종사하는 분들도 이렇게 열심히 개발하시는 것을 보면 블록체인을 공부하고 있지만 나중엔 결국 인공지능과도 융합된 개발을 해야 하지 않을까라는 생각이 들었고 블록체인의 기초 개념이 어느 정도 자리가 잡히면 인공지능에 대해서도 공부해 볼 생각이다.





참! 행사를 시작하면서 참가 사은품(?)으로 보조배터리를 받기도 했다. 안 그래도 보조배터리를 새로 살까 고민하고 있었는데 C타입도 지원하고 괜찮은 걸 주신 것 같다.







어쨌든 같은 연구실 연구원들과 함께 돌아다니며 몇몇 발표 행사에 참석해 학생분들이 쓴 논문과 교수님께서 직접 해주시는 강의 형식의 수업을 듣기도 했다.

이번에 참석한 발표 행사는 다음과 같다.


  1. 블록체인의 핵심 개념, 산업별 응용 사례 그리고 소프트웨어 공학 연구 이슈 소개 (전북대 김순태 교수님)

  2. 계약 관련 음성 녹취 정보의 신뢰성 있는 저장과 체계적 관리를 위한 블록체인 기반 시스템 구성 방식 (김인근 연구원)

  3. 그룹 리더 선출을 통한 비트코인 병렬 트랜잭션 처리 (이도현 연구원)

  4. 블록체인 기반 실시간 수하물 트래킹 시스템 (이열국 연구원)


이 중에서 아래 3개는 BaSE 연구실의 연구원들이 발표한 논문이었다.

아직 모든 것을 이해하기에는 부족하지만 연구실 연구원들처럼 좋은 아이디어를 생각해 내고, 또 다른 공식적인 자리에서 발표할 수 있는 기회가 생긴다면 한 번 도전해보고 싶다.











이번 글은 하나에 몰아서 정리하긴 했지만 실제로는 3일에 걸쳐서 진행한 행사였다.

마지막으로, 두 번째 날 저녁에는 모든 사람들이 한 곳에 모여서 만찬을 즐기기도 했다.







전체적으로 블록체인뿐만 아니라 소프트웨어 연구 동향을 알 수 있는 좋은 기회였고, 같은 연구실에 있는 연구원들과도 조금 더 가까워질 수 있는 시간이었던 것 같아서 너무 만족스럽다.
